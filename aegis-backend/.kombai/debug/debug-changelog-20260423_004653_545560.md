## Changes Made

### 1. Fixed CORS wildcard + credentials conflict
- **File:** `aegis-backend/app/main.py`
- **Change:** Line 70 — replaced `allow_origins=["*"]` with `allow_origins=settings.CORS_ORIGINS`
- **Why:** `allow_origins=["*"]` combined with `allow_credentials=True` violates the CORS spec. Browsers reject responses that carry `Access-Control-Allow-Origin: *` alongside `Access-Control-Allow-Credentials: true`. The correct origins were already defined in `settings.CORS_ORIGINS` (default: `http://localhost:3000,http://localhost:8000,...`) but were never being used.
- **Revert:** Change `allow_origins=settings.CORS_ORIGINS` back to `allow_origins=["*"]`

### 2. Fixed WebSocket endpoint path mismatch
- **Files:** `aegis-backend/app/api/router.py`, `aegis-backend/app/main.py`
- **Change (router.py):** Removed `websocket` from the imports and removed `api_router.include_router(websocket.router, ...)` call. The WebSocket router was incorrectly included inside `api_router` (which carries the `/api` prefix), placing the endpoint at `/api/ws/{session_id}`.
- **Change (main.py):** Added `app.include_router(ws_routes.router, tags=["WebSocket"])` directly on the app (no `/api` prefix), placing the endpoint at `/ws/{session_id}`.
- **Why:** The frontend `websocket-client.ts` connects to `ws://localhost:8000/ws/{session_id}` and the Next.js rewrite maps `/ws/:path*` → backend `/ws/:path*`. Both expect the endpoint at `/ws/{session_id}`, but it was being served at `/api/ws/{session_id}` causing all WebSocket connections to fail with 404.
- **Revert:**
  - `router.py`: Re-add `websocket` to imports and add `api_router.include_router(websocket.router, tags=["WebSocket"])` at the end
  - `main.py`: Remove the `from app.api.routes import websocket as ws_routes` and `app.include_router(ws_routes.router, tags=["WebSocket"])` lines
